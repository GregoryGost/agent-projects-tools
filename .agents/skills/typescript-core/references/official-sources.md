# TypeScript Core Official Sources

Use these sources when TypeScript language behavior, compiler options, class behavior, or type-system details need verification.

## Official sources

- TypeScript Docs: https://www.typescriptlang.org/docs/
- TypeScript Handbook: https://www.typescriptlang.org/docs/handbook/intro.html
- TypeScript Handbook: Classes: https://www.typescriptlang.org/docs/handbook/2/classes.html
- TypeScript TSConfig Reference: https://www.typescriptlang.org/tsconfig/
- TypeScript TSConfig: `strictPropertyInitialization`: https://www.typescriptlang.org/tsconfig/#strictPropertyInitialization
- TypeScript TSConfig: `noImplicitOverride`: https://www.typescriptlang.org/tsconfig/#noImplicitOverride
- TypeScript Utility Types: https://www.typescriptlang.org/docs/handbook/utility-types.html
- MDN JavaScript Classes: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Classes

## Best-practice references

- Google TypeScript Style Guide: https://google.github.io/styleguide/tsguide.html
- typescript-eslint `no-extraneous-class`: https://typescript-eslint.io/rules/no-extraneous-class/
- typescript-eslint `prefer-readonly`: https://typescript-eslint.io/rules/prefer-readonly/

Project code style, formatting, linting, module format, and validation commands come from `CODEX_PROJECT.md` and the target repository.
