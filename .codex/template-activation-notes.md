# Template Activation Notes

This note explains how to use `.codex/project.template.md`.

- The template is intentionally broad so it can support different project types.
- The final `CODEX_PROJECT.md` must be narrow and project-specific.
- Keep only sections, profiles, rules, skills, commands, and integrations that apply to the concrete repository.
- Remove unused optional sections from the final `CODEX_PROJECT.md`.
- If the project type or active stack cannot be determined from repository files, ask the user before creating `CODEX_PROJECT.md`.
